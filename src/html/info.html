Información de la herramienta

Esta herramienta permite, a partir de dos diagramas con formato .bpmn, dos .csv con los nombres de las tareas y una matriz de coincidencia en formato .csv, realizar una comparativa gráfica entre ellos.
Los diagramas .bpmn deben tener las tareas con identificadores asignados de la siguiente manera:
Por ejemplo, para una tarea con nombre "Obtain-all conference", su identificador asociado debe utilizar camelCase y considerar una palabra toda secuencia de caracteres hasta encontrar un espacio. Así, su identificador será "obtain-allConference".
En caso de que dos tareas del mismo diagrama tengan el mismo nombre, se añadirá un número al final del identificador de la tarea, de forma que no se repitan.
Por ejemplo, si tenemos dos tareas con nombre "Obtain-all conference", sus identificadores asociado serán "obtain-allConference" y "obtain-allConference2".

Por favor, utilice diagramas en formato .bpmn cuyo nombre no contenga caracteres especiales ni espacios.
Escoja los diagramas desde la carpeta del proyecto según se indica a continuación:

crs-get-conference-bpmn.bpmn junto con crs-update-conference-bpmn.bpmn
crs-delete-paper-bpmn.bpmn junto con crs-delete-subject-bpmn.bpmn

  2. Escoja entre las combinaciones de diagrmas bpmn mencionadas anteriormente.

  5. Una vez pulsado sobre "OK", vuelva a la página de inicio para ver el resultado.

Interpretación

A continuación, se expresa el significado de cada una de las cuatro opciones de comparativa soportadas por la herramienta:

  • Mínimo. Realiza una comparativa gráfica para cada tarea del diagrama A respecto con cada tarea del diagrama B, obteniendo la mínima coincidencia en cada caso.
    Solo utiliza el color rojo para remarcar las coincidencias encontradas.
  • Mínimo especial. Igual que Mínimo, pero utiliza verde, amarillo y rojo para remarcar las coincidencias encontradas.
  • Máximo. Realiza una comparativa gráfica para cada tarea del diagrama A respecto con cada tarea del diagrama B, obteniendo la máxima coincidencia en cada caso.
    Solo utiliza el color verde para remarcar las coincidencias encontradas.
  • Máximo especial. Igual que Máximo, pero utiliza verde, amarillo y rojo para remarcar las coincidencias encontradas.
En los modos Mínimo especial y Máximo especial, los colores se pueden interpretar como se expone a continuación:
  • Verde: alta coincidencia de la tarea del diagrama A respecto con la tarea del diagrama B.
  • Amarillo: coincidencia media de la tarea del diagrama A respecto con la tarea del diagrama B.
  • Rojo: baja coincidencia de la tarea del diagrama B respecto con la tarea del diagrama B.

    • Se utiliza una primera coincidencia, es decir, si una tarea del diagrama A tiene el mismo grado de coincidencia con dos o más tareas del diagrama B, se colorea solo la primera coincidencia. Esto se hace con motivo de no sobrecargar el diagrama con colores, facilitar su lectura e interpretación.

Information about the tool

This tool allows, from two diagrams in .bpmn format, two .csv with the task names and a matching matrix in .csv format, to make a graphical comparison between them.
The .bpmn diagrams should have the tasks with identifiers assigned as follows:
For example, for a task named "Obtain-all conference", its associated identifier should use camelCase and consider a word any sequence of characters until a space is found. Thus, its identifier will be "obtain-allConference".
In case two tasks in the same diagram have the same name, a number will be added at the end of the task identifier, so that they are not repeated.
For example, if we have two tasks with the name "Obtain-all conference", their associated identifiers will be "obtain-allConference" and "obtain-allConference2".

Please, use diagrams in .bpmn format, whose name does not contain special characters or spaces.

Usage mode

Below is a summary of the steps to follow in order to use the tool:

  1. Go to the "Select diagrams" page.
  2. Choose from the above mentioned bpmn diagram combinations.
  3. Select a comparison option after you have chosen the diagrams.
  4. Click the "OK" button if you are sure of your choice. Otherwise, click on "Clear" and return to step 2.
  5. Once you have clicked on "OK", return to the home page to see the result.

Interpretation

The following is the meaning of each of the four comparison options supported by the tool:

  • Minimum. Performs a graphical comparison for each task in diagram A with respect to each task in diagram B, obtaining the minimum coincidence in each case.
    It only uses red color to highlight the matches found.
  • Special minimum. Same as Minimum, but uses green, yellow and red to highlight the matches found.
  • Maximum. Performs a graphical comparison for each task in diagram A with respect to each task in diagram B, obtaining the maximum match in each case.
    It only uses the green color to highlight the matches found.
  • Special maximum. Same as Maximum, but uses green, yellow and red to highlight the matches found.
In the Special Minimum and Special Maximum modes, the colors can be interpreted as follows:
  • Green: high match of the task in diagram A with respect to the task in diagram B.
  • Yellow: medium match of the task in diagram A with respect to the task in diagram B.
  • Red: low match of the task in diagram B with respect to the task in diagram B.

    • A first match approachis is used, that is, if a task in diagram A has the same degree of coincidence with two or more tasks in diagram B, only the first match is colored. This is done in order not to overload the diagram with colors and to facilitate its reading and interpretation.

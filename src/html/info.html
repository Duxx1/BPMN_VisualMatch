Información de la herramienta

Esta herramienta permite, a partir de dos diagramas con formato .bpmn, dos .csv con los nombres de las tareas, una matriz de coincidencia en formato .csv y dos ficheros .csv con los pools de las tareas, realizar una comparativa gráfica entre ellos.
Esta herramienta se basa en WordDomain, de donde se extraen la mayoría de ficheros necesarios.
Se debe tener en cuenta que esta herramienta solo funciona con WordDomain, por lo que se tendrá que haber hecho una comparación previa entre dos diagramas si se desean luego visualizar en esta herramienta, y se deberá seguir el manual de la documentación.

Por favor, utilice diagramas en formato .bpmn cuyo nombre no contenga caracteres especiales ni espacios. Lo mismo con el resto de archivos.
Escoja los diagramas desde la carpeta bpmn_diagrams del proyecto, según se indica a continuación:

crs-get-conference-bpmn.bpmn junto con crs-update-conference-bpmn.bpmn
crs-create-article-bpmn.bpmn junto con crs-delete-track-bpmn.bpmn

IMPORTANTE

Use los diagramas mencionados arriba y en ese orden para visualizar la comparativa gráfica.

Si desea añadir nuevos diagramas, siga las instrucciones descritas en la documentación de la herramienta.

Modo de uso

A continuación, se presenta un resumen de los pasos a seguir para utilizar la herramienta.

Si desea realizar una visualización de un diagrama sin ningún tipo de comparativa, haga lo siguiente:

  1. Diríjase a la página de "Select Diagrams".
  2. Seleccione el archivo del diagrama desde la carpeta bpmn_diagrams del proyecto, en la sección indicada.
  3. Pulse sobre el botón "VIEW DIAGRAM".

Si lo que desea es visualizar la comparativa entre dos diagramas, los pasos a seguir son los siguientes:

  1. Diríjase a la página de "Select Diagrams".
  2. Escoja entre las combinaciones de diagramas bpmn mencionadas anteriormente desde la carpeta bpmn_diagrams.
  3. Seleccione una opción de comparación tras haber escogido los diagramas.
  4. Pulse el botón de "OK" si está seguro de su elección. En caso contrario, pulse sobre "Limpiar" y vuelva al paso 2.
  5. Una vez pulsado sobre "OK", diríjase a la página "Compare" para ver el resultado.

Interpretación

A continuación, se expresa el significado de cada una de las cuatro opciones de comparativa soportadas por la herramienta:

  • Mínimo. Realiza una comparativa gráfica para cada tarea del diagrama A respecto con cada tarea del diagrama B, obteniendo la mínima coincidencia en cada caso.
    Solo utiliza el color rojo para remarcar las coincidencias encontradas.
  • Mínimo especial. Igual que Mínimo, pero utiliza verde, amarillo y rojo para remarcar las coincidencias encontradas.
  • Máximo. Realiza una comparativa gráfica para cada tarea del diagrama A respecto con cada tarea del diagrama B, obteniendo la máxima coincidencia en cada caso.
    Solo utiliza el color verde para remarcar las coincidencias encontradas.
  • Máximo especial. Igual que Máximo, pero utiliza verde, amarillo y rojo para remarcar las coincidencias encontradas.
En los modos Mínimo especial y Máximo especial, los colores se pueden interpretar como se expone a continuación:
  • Verde: alta coincidencia de la tarea del diagrama A respecto con la tarea del diagrama B.
  • Amarillo: coincidencia media de la tarea del diagrama A respecto con la tarea del diagrama B.
  • Rojo: baja coincidencia de la tarea del diagrama B respecto con la tarea del diagrama B.

    • Se utiliza una primera coincidencia, es decir, si una tarea del diagrama A tiene el mismo grado de coincidencia con dos o más tareas del diagrama B, se colorea solo la primera coincidencia. Esto se hace con motivo de no sobrecargar el diagrama con colores, facilitar su lectura e interpretación.

Information about the tool

This tool allows, from two diagrams in .bpmn format, two .csv with the task names, a matching matrix in .csv format and two .csv files with the tasks pools, to make a graphical comparison between them.
This tool is based on WordDomain, from where most of the necessary files are extracted.
It must be taken into account that this tool only works with WordDmain, so you will have to have made a previous comparison between two diagrms if you want to visualizar them in this tool, and you will have to follow the documentation manual.

Please, use diagrams in .bpmn format, whose name does not contain special characters or spaces. The same with the rest of the files.
Choose the diagrams from the project folder bpmn_diagrams as indicated below:

crs-get-conference-bpmn.bpmn in conjunction with crs-update-conference-bpmn.bpmn
crs-create-article-bpmn.bpmn junto con crs-delete-track-bpmn.bpmn

IMPORTANT

Use the above diagrams in that order to visualize the graphical comparison.

If you want to add new diagrams, folow the instructions described in the tool documentation.

Usage mode

Below is a summary of the steps to follow in order to use the tool.

If you want to visualize a diagram without any comparison, do the following:

  1. Go to "Select Diagrams" page.
  2. Select the the diagram file from the bpmn_diagrams folder of the project, in the indicated section.
  3. Click on the "VIEW DIAGRAM" button.

If you want to visualize the comparison between two diagrams, the steps to follow are as indicated below:

  1. Go to the "Select diagrams" page.
  2. Choose from the above mentioned bpmn diagram combinations.
  3. Select a comparison option after you have chosen the diagrams from the bpmn_diagrams folder of the project.
  4. Click the "OK" button if you are sure of your choice. Otherwise, click on "Clear" and return to step 2.
  5. Once you have clicked on "OK", go to the "Compare" page to see the result.

Interpretation

The following is the meaning of each of the four comparison options supported by the tool:

  • Minimum. Performs a graphical comparison for each task in diagram A with respect to each task in diagram B, obtaining the minimum coincidence in each case.
    It only uses red color to highlight the matches found.
  • Special minimum. Same as Minimum, but uses green, yellow and red to highlight the matches found.
  • Maximum. Performs a graphical comparison for each task in diagram A with respect to each task in diagram B, obtaining the maximum match in each case.
    It only uses the green color to highlight the matches found.
  • Special maximum. Same as Maximum, but uses green, yellow and red to highlight the matches found.
In the Special Minimum and Special Maximum modes, the colors can be interpreted as follows:
  • Green: high match of the task in diagram A with respect to the task in diagram B.
  • Yellow: medium match of the task in diagram A with respect to the task in diagram B.
  • Red: low match of the task in diagram B with respect to the task in diagram B.

    • A first match approachis is used, that is, if a task in diagram A has the same degree of coincidence with two or more tasks in diagram B, only the first match is colored. This is done in order not to overload the diagram with colors and to facilitate its reading and interpretation.
